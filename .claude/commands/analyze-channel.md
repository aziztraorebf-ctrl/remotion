## Context
- Projet : Video YouTube educative animee avec Remotion
- But : Extraire des learnings concrets d'une chaine pour ameliorer notre pipeline
- Outil : `yt-dlp` installe via brew (telecharge videos + sous-titres)
- Dossier references : `research/references/`

## Your Task

Analyser une chaine YouTube fournie par Aziz pour en extraire les patterns de style, retention et strategie applicables a notre projet.

### Step 1: Recevoir le brief

Demander a Aziz :
- Lien(s) YouTube (video ou chaine)
- Pourquoi cette chaine l'interesse (angle specifique ?)
- Combien de videos analyser (defaut : 3-5 representatives)

### Step 2: Telecharger et extraire

```bash
# Creer dossier pour cette chaine
mkdir -p research/references/<nom-chaine>/

# Telecharger video + sous-titres (pas la video HD, juste audio + subs)
yt-dlp --write-sub --write-auto-sub --sub-lang fr,en --skip-download -o "research/references/<nom-chaine>/%(title)s.%(ext)s" <URL>

# Si analyse visuelle necessaire, telecharger en basse qualite
yt-dlp -f "worst[ext=mp4]" -o "research/references/<nom-chaine>/%(title)s.%(ext)s" <URL>
```

### Step 3: Analyser chaque video

Pour chaque video, produire une fiche structuree :

```markdown
## [Titre de la video]
- **Duree** : X min
- **Vues** : X (ratio vues/abonnes)
- **Date** : YYYY-MM-DD

### Hook (0-30s)
- Technique utilisee (stat choc, question, teaser visuel, confession)
- Temps avant le premier "pourquoi rester" implicite
- Visuel : que voit-on exactement ?

### Structure narrative
- Type (chronologique, probleme-solution, mystere-revelation, jeu video)
- Nombre d'actes/segments
- Transitions entre segments (hard cut, phrase-pont, visuel)

### Style visuel
- Type d'animation (pixel art, motion graphics, sketch, AI, mixte)
- Changements de scene : frequence (toutes les X secondes)
- Data visualization : presente ? style ?
- Personnages : comment representes ?

### Audio
- Voix : ton, rythme, pauses, emphasis
- Musique : presence, volume relatif, style
- SFX : lies aux visuels ou ambiance ?

### Retention estimee
- Moments de drop potentiel (longues explications, transitions molles)
- Moments forts (revelations, visuels saisissants, twists)

### Learnings applicables
- [ ] Ce qu'on peut reproduire dans notre pipeline
- [ ] Ce qui necessite des outils qu'on n'a pas
- [ ] Ce qui ne correspond pas a notre style
```

### Step 4: Synthese cross-videos

Apres avoir analyse toutes les videos :

```markdown
## Synthese : [Nom de la chaine]

### Statistiques generales
- Abonnes : X | Videos : X | Frequence : X/mois
- Vues moyennes : X | Ratio engagement : X%

### Pattern dominant
[Quel est le "secret" de cette chaine en 2-3 phrases]

### Ce qu'on integre (ACTION)
1. [Learning concret + comment l'appliquer]
2. [...]

### Ce qu'on note (CONSIDERATION)
1. [Interesting mais pas urgent]

### Ce qu'on rejette (REJETE)
1. [Pourquoi ca ne s'applique pas a nous]
```

### Step 5: Mise a jour du projet

- Sauvegarder synthese dans `research/references/<nom-chaine>/ANALYSIS.md`
- Si des learnings impactent le pipeline : proposer les modifications
- Si des learnings impactent le script writing : mettre a jour le skill

### Guardrails

- NE PAS copier le style d'une chaine. Extraire les PRINCIPES, pas les details
- Toujours contextualiser : "cette chaine a X abonnes, dans la niche Y, cela marche pour elle parce que Z"
- Si Aziz est enthousiaste, rester factuel : valider ce qui est reproductible vs ce qui est specifique a cette chaine
- Comparer avec ce qu'on sait deja (Last 30 Days scans, persona, style guide)
